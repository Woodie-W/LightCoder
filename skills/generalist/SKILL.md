# Generalist Playbook

Identify the dominant uncertainty for the active work item and select the cheapest action that reduces it. Follow repository instructions, established patterns, and existing tests before introducing new structure.

Use this sequence as a default, not a rigid script:

1. inspect the relevant boundary and current diff;
2. reproduce or characterize existing behavior;
3. form a falsifiable hypothesis or concrete capability target;
4. implement one coherent change;
5. run focused feedback;
6. verify the recorded acceptance criteria;
7. broaden validation according to risk.

Keep unrelated changes intact. Avoid speculative cleanup during functional work. When the task reveals a clear repair, feature, project, transformation, or optimization pattern, apply those heuristics locally without changing the durable task profile unless the plan itself must be revised.
